# Temporary Docs Handoff: TemplateRunner Calendar Targets

This temporary note summarizes the `np.Templating` 2.4.0 updates for documentation work. It can be removed after the public docs are updated.

## Summary

`TemplateRunner` frontmatter field `getNoteTitled` can now target calendar notes more directly.

Previously, `getNoteTitled` handled a few special targets such as `<today>`, `<thisweek>`, `<nextweek>`, `<current>`, `<choose>`, and `<select>`. Daily dates rendered from template tags, such as `2026-05-03`, could fall through to regular project-note title lookup.

Now, after template tags in `getNoteTitled` are rendered, valid calendar note titles route to calendar notes instead of regular note title lookup.

## Supported Calendar Title Formats

These rendered values are treated as calendar note targets:

- Daily: `YYYY-MM-DD`, for example `2026-05-03`
- Monthly: `YYYY-MM`, for example `2026-06`
- Quarterly: `YYYY-Qn`, for example `2026-Q3`
- Yearly: `YYYY`, for example `2026`

Invalid date-shaped values are not treated as calendar notes. For example, `2024-99-99`, `2024-13`, and `2024-Q5` are rejected by strict validation and continue through the normal handling path.

## New Relative Tokens

`getNoteTitled` now supports these additional relative calendar tokens:

- `<tomorrow>`
- `<yesterday>`
- `<thismonth>`
- `<nextmonth>`
- `<thisquarter>`
- `<nextquarter>`
- `<thisyear>`
- `<nextyear>`

Existing tokens remain supported:

- `<today>`
- `<thisweek>`
- `<nextweek>`
- `<current>`
- `<choose>`
- `<select>`

## Examples

Write to tomorrow's daily note:

```yaml
getNoteTitled: <tomorrow>
```

Write to a date generated by a template expression:

```yaml
getNoteTitled: <%- date.tomorrow('YYYY-MM-DD') %>
```

Write to next month's monthly note:

```yaml
getNoteTitled: <nextmonth>
```

Write to a rendered monthly note title:

```yaml
getNoteTitled: <%- date.now('YYYY-MM', '1m') %>
```

Write to next quarter's quarterly note:

```yaml
getNoteTitled: <nextquarter>
```

Write to a rendered yearly note title:

```yaml
getNoteTitled: 2026
```

## Boundary Behavior

Relative tokens are calculated with calendar-aware date math:

- On `2024-12-31`, `<nextmonth>` resolves to `2025-01`.
- On `2024-12-31`, `<nextquarter>` resolves to `2025-Q1`.
- On `2024-12-31`, `<nextyear>` resolves to `2025`.

Month, quarter, and year notes use NotePlan calendar note formats (`YYYY-MM`, `YYYY-Qn`, `YYYY`) and open/write using the corresponding calendar timeframe.

## Documentation Notes

When documenting rendered template expressions, prefer explicit output formats:

```yaml
getNoteTitled: <%- date.tomorrow('YYYY-MM-DD') %>
```

Avoid relying on `date.tomorrow()` without a format when the target must be a calendar note, because the user's default templating date format may not be `YYYY-MM-DD`.
